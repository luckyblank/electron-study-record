## 🔄 数据迁移行为说明

对于现有用户：
1. 首次启动新版本，自动检测旧数据库版本（v1）
2. 自动备份 → 增量执行 v1→v2→v3→v4 迁移
3. 历史数据完整保留：
   - 已有 study_sessions 记录不动
   - 自动计算并回填 `total_study_seconds` 和 `total_sessions`
   - 自动添加 `note` 字段（默认空）
   - 预置 4 个默认标签
4. 若迁移失败 → 自动从备份恢复，应用以旧版本结构运行

## ⚠️ 已知注意事项

1. **首次升级耗时**：v3 迁移会扫描全部历史 session 计算累计，数据量大时可能略有等待（一般 <100ms）
2. **IDE 提示 `event` 未使用**：Electron IPC handler 的标准签名，无害
3. **`app-region` CSS 警告**：Electron 专属属性，IDE 误报
4. **历史删除现在有 `confirm()` 弹窗**：避免误删（原版无确认）
5. **系统通知优先于页面 toast**：与原版行为一致，但 `warning` 类型只走页面 toast
