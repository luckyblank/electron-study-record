
# 学习宠物系统 - 完整实施计划

---

## 一、Context：为什么做这个功能

### 需求背景
用户希望在主页面增加一个成长型宠物系统，让学习过程更有陪伴感和正向激励。宠物会随着学习时间成长，不同的学习状态会触发对应的互动动画，让枯燥的学习记录变得生动有趣。

### 核心价值
- **陪伴感**：学习时有"小伙伴"一起，减少孤独感
- **正向反馈**：每段学习结束都能看到宠物成长，激励持续学习
- **状态可视化**：通过宠物表情/动作直观反映当前学习状态
- **收集乐趣**：多种宠物可解锁，增加长期使用动力

### 预期效果
1. 开始/暂停/结束学习时，宠物有对应的情绪反馈
2. 学习时长转化为宠物经验值，支持升级和解锁新宠物
3. 点击宠物可以互动，增加亲密度
4. 所有状态持久化保存，重启应用不丢失

## 二、推荐实施架构

### 设计原则
1. **零额外依赖**：纯 CSS/SVG + JavaScript，不引入新库
2. **架构复用**：完全融入现有 IPC/user_config 模式，参考徽章系统
3. **渐进增强**：宠物功能不影响核心计时逻辑，可独立开关
4. **主题适配**：所有视觉元素兼容 light/dark/eyecare 三种主题

## 三、宠物属性系统设计

### 宠物元数据

| ID | 名称 | 图标 | 主题色 | 解锁条件 |
|----|------|------|--------|----------|
| `cat` | 学霸猫 | 🐱 | `#FF9800` | 默认解锁 |
| `dog` | 努力汪 | 🐕 | `#4CAF50` | 累计学习 10 小时 或 连续 7 天 |
| `owl` | 智慧鸮 | 🦉 | `#9C27B0` | 获得 5 个徽章 或 累计 50 小时 |

### 宠物状态枚举

```javascript
const PET_STATES = {
  IDLE: 'idle',           // 待机
  STUDYING: 'studying',   // 学习中
  PAUSED: 'paused',       // 暂停中
  CELEBRATE: 'celebrate', // 结束庆祝
  TIRED: 'tired',         // 疲惫提醒
  LEVEL_UP: 'levelup'     // 升级中
}
```

### 核心属性算法

| 属性 | 说明 | 更新规则 | 范围 |
|------|------|----------|------|
| **等级 Level** | 宠物等级 | 经验值累积升级 | 1-100 |
| **经验 EXP** | 学习积累的经验 | 学习 1 分钟 = 1 EXP | 0-∞ |
| **心情 Mood** | 当前情绪值 | 连续学习+1/分钟，暂停-0.5/分钟 | 0-100 |
| **亲密度 Affection** | 与用户的羁绊 | 点击互动 +1（每日上限 20 次） | 0-100 |
| **能量 Energy** | 学习精力值 | 学习-1/分钟，休息+2/分钟 | 0-100 |

### 等级经验公式

```
升级所需 EXP = 100 + (level - 1) * 50

示例：
L1 → L2: 100 EXP (约 1.67 小时)
L1 → L10: 850 EXP (约 14 小时)
L1 → L50: 6250 EXP (约 104 小时)
L1 → L100: 15850 EXP (约 264 小时)
```

### 状态动画矩阵

| 触发事件 | 宠物状态 | 动画效果 | CSS 实现 |
|----------|----------|----------|----------|
| 应用启动 | IDLE | 缓慢呼吸 + 随机眨眼 | `@keyframes breathe` + `@keyframes blink` |
| 开始学习 | STUDYING | 身体前倾 + 眼睛睁大 + 尾巴快摇 | `transform: scale(1.05)` + 尾巴摆动 |
| 暂停学习 | PAUSED | 歪头疑惑 + 眨眼变慢 + 尾巴静止 | `transform: rotate(-5deg)` + 半闭眼 |
| 结束学习 | CELEBRATE | 向上跳跃 + 撒星星粒子 + 经验飘字 | `@keyframes jumpBounce` + DOM 粒子 |
| 达到阈值 | LEVEL_UP | 发光光环脉冲 + 弹跳 + 升级数字飘出 | `box-shadow` 脉冲 + `@keyframes floatUp` |
| 超2小时 | TIRED | 身体下垂 + 眼睛半睁 + 叹气气泡 | `transform: translateY(2px)` + 叹气动画 |
| 点击互动 | INTERACT | 身体微动 + 红心飘出 + 开心表情 | `@keyframes heartFloat` |

### 粒子效果复用
参考现有徽章解锁/Toast 动画模式，使用动态创建 DOM + CSS keyframes，不引入 Canvas。

## 五、状态触发与集成点

### 核心事件注入点

```javascript
// ─── renderer.js 现有函数增强 ───

async function startSession() {
  // ... 原有逻辑 ...
  const id = await window.studyRecord.startSession(...)
  
  // ↓ 新增：宠物状态切换
  await petManager.setState(PET_STATES.STUDYING)
  petManager.playAnimation('excited')
  petManager.startEnergyDrain()
  // ↑
  
  return id
}

async function pauseSession() {
  // ... 原有逻辑 ...
  await window.studyRecord.pauseSession(...)
  
  // ↓ 新增
  await petManager.setState(PET_STATES.PAUSED)
  petManager.playAnimation('confused')
  petManager.startEnergyRecovery()
  // ↑
}

async function resumeSession() {
  // ... 原有逻辑 ...
  await window.studyRecord.resumeSession(...)
  
  // ↓ 新增
  await petManager.setState(PET_STATES.STUDYING)
  petManager.playAnimation('excited')
  petManager.startEnergyDrain()
  // ↑
}

async function endSession() {
  // ... 原有逻辑 ...
  const result = await window.studyRecord.endSession(...)
  const sessionSeconds = result.duration
  
  // ↓ 新增：结算经验与动画
  await petManager.setState(PET_STATES.CELEBRATE)
  const earnedExp = Math.floor(sessionSeconds / 60)
  const didLevelUp = await petManager.addExp(earnedExp)
  
  petManager.playAnimation('celebrate', { exp: earnedExp })
  if (didLevelUp) {
    setTimeout(() => petManager.playAnimation('levelup'), 2500)
  }
  setTimeout(() => petManager.setState(PET_STATES.IDLE), 5000)
  // ↑
  
  return result
}
```

### 长时学习疲惫提醒
在计时器 tick 中监控，超过 2 小时触发宠物疲惫提示（但不强制打断学习）：
```javascript
// 在 1 秒更新计时器回调中
function updateTodayDisplay() {
  // ... 原有逻辑 ...
  if (currentSessionStartTime && !isPaused) {
    const elapsed = Date.now() - currentSessionStartTime - currentSessionPausedDuration
    if (elapsed > 2 * 60 * 60 * 1000 && !petTiredWarnShown) {
      petManager.showTiredWarning()
      petTiredWarnShown = true
    }
  }
}
```

| IPC Handler | `main.js` | 宠物状态读写、解锁检查 |
| 桥接层 | `preload.js` | 暴露 pet:* 系列 API |

## 风险与回退

### 潜在风险
1. **SVG 动画性能**：多 CSS 动画同时运行可能影响低端设备
   - 缓解：使用 `will-change` 和 `transform` 开启硬件加速

2. **状态持久化失败**：宠物状态丢失可能打击用户积极性
   - 缓解：每次状态变更立即持久化，同时保留默认状态兜底

3. **宠物形象不符合预期**：用户可能不喜欢当前设计
   - 缓解：宠物系统默认开启，未来可扩展更多形象/皮肤

### 可选开关
如果用户希望关闭宠物功能，可在设置中添加一个简单的显示/隐藏 toggle，不影响任何核心逻辑。
