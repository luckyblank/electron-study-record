# 学习时间记录应用 - 优化改进计划

## Context

这是一个基于 Electron + SQLite3 的学习时间记录桌面应用，当前版本功能完整但存在以下问题：
- 代码层面：有已知 bug、错误处理不完善、性能和可维护性可提升
- UI 层面：设计风格较保守，缺少现代动效和数据可视化
- 功能层面：缺少激励机制、数据导出、标签分类等实用功能

本计划旨在分阶段优化项目，从代码质量修复到 UI 升级，再到功能扩展，保持应用轻量特性的同时提升用户体验和功能价值。

## 优化原则

1. **先修复后增强**：先解决已知 bug 和数据安全问题，再添加新功能
2. **保持轻量**：不引入重型框架，使用原生 HTML/CSS/JS + 轻量级库
3. **渐进式**：每个阶段可独立交付，不影响现有功能使用
4. **向后兼容**：数据库升级不破坏现有数据，支持平滑迁移

## Phase 1: 代码质量修复（优先级：高）

**目标**：修复已知 bug，提升代码稳定性和可维护性

### 1.1 关键 Bug 修复

**文件**：`main.js` - `initWord()` 函数
**问题**：打包应用检测块中的 `path.join()` 结果没有赋值给 `txtPath`
**修复**：修正条件分支的路径赋值

**文件**：`main.js` - `safeQuit()` 函数  
**问题**：固定 500ms 延迟不可靠，IPC 异步写入可能未完成就退出
**修复**：添加渲染进程确认机制，session 结束后再退出

### 1.2 性能优化

**文件**：`main.js` - 数据库初始化
**优化**：为 `study_sessions.start_time` 添加索引，提升按时间范围查询性能
```sql
CREATE INDEX IF NOT EXISTS idx_study_sessions_start_time ON study_sessions(start_time);
```

**文件**：`renderer.js` - 定时器管理
**优化**：添加 `beforeunload` 事件清理定时器，降低历史刷新频率从 30s 到 2min

### 1.3 错误处理增强

**文件**：`preload.js`
**优化**：为所有 IPC 调用添加超时包装（5秒超时）

**文件**：`main.js` - 数据库 handlers
**优化**：错误时正确 reject Promise，而非静默返回 undefined

**文件**：`renderer.js` - Session 操作
**优化**：添加原子锁防止快速双击导致的竞态条件

### 1.4 代码清理

**文件**：`renderer.js`
**优化**：提取 `renderHistoryItem()` 通用函数，消除两处重复的历史渲染逻辑

**文件**：全局
**优化**：提取魔法数字为常量配置（2小时秒数、刷新间隔、退出延迟等）

**文件**：`index.html` + `styles.css`
**优化**：将所有内联样式移至 CSS 文件，使用类名控制

## Phase 2: UI/视觉升级（优先级：高）

**目标**：遵循 web-design-engineer 设计原则，打造现代、简洁、有动效的界面

### 2.1 图标系统现代化

### 2.2 CSS 变量与主题系统

**文件**：`styles.css`
- 重构为 CSS 变量驱动的色彩系统
- 实现玻璃拟态效果（毛玻璃背景 + 模糊 + 边框）
- 预设 3 套主题：
  - 默认浅色（当前效果）
  - 暗夜模式（深色背景 + 高对比度）
  - 护眼绿（低饱和度绿色系）

**文件**：`renderer.js` + `main.js` + `preload.js`
- 添加 IPC 通道读写主题配置
- 实现主题切换按钮和实时预览

### 2.3 动效与微交互

**文件**：`styles.css` + `renderer.js`
- 按钮点击效果：按下缩小 + 释放回弹
- 面板切换：展开/收起的平滑过渡动画
- 时间数字：秒数变化时的轻微跳动效果
- 学习中状态：时间旁的呼吸式指示灯
- 里程碑庆祝：达到 1h/2h/4h 时的粒子效果

### 2.4 布局优化

**文件**：`index.html` + `styles.css`
- 标题栏添加最小化/关闭按钮（透明风格）
- 右侧菜单栏触发区域从 30px 扩大到可拖拽显示
- 历史记录卡片添加悬停效果和深度层次

## Phase 3: 数据可视化增强（优先级：中）

**目标**：从简单的列表展示升级为直观的数据可视化

### 3.1 迷你进度环

**位置**：今日学习时间旁
**功能**：
- 环形进度条显示今日目标完成度
- 中心显示百分比
- 颜色随进度渐变（红→橙→绿→蓝→紫）
**技术**：SVG `<circle>` + `stroke-dasharray` 实现

### 3.2 周趋势柱状图

**位置**：历史弹窗顶部
**功能**：
- 7 根柱子代表最近 7 天学习时长
- 悬停显示精确数值和同比变化
**技术**：CSS Grid + div 纯 CSS 实现

### 3.3 学习热力图

**位置**：设置面板可展开区域
**功能**：
- GitHub 风格的 7x5 贡献图（最近 35 天）
- 颜色深浅代表学习时长
- 悬停显示日期和具体时长
**技术**：CSS Grid + 动态生成的 div 元素

### 3.4 智能托盘菜单

**文件**：`main.js` - `createTray()`
**功能**：
- 实时显示今日已学习时长
- 根据状态显示"开始学习"/"结束学习"菜单项
- 快速查看今日详情入口

## Phase 4: 功能扩展 - 激励系统（优先级：高）

**目标**：从被动记录工具升级为主动激励的学习助手

### 4.1 目标设定系统

**数据库变更**：`user_config` 表新增键
- `daily_goal_seconds` - 每日目标（默认 7200 秒 = 2 小时）
- `weekly_goal_seconds` - 每周目标

**UI**：
- 设置面板添加目标设定入口
- 进度环实时显示完成度
- 目标达成时的庆祝动画和系统通知

### 4.2 连续打卡机制

**数据库变更**：`user_config` 表新增键
- `last_study_date` - 最后学习日期
- `current_streak` - 当前连续天数
- `max_streak` - 历史最长连续天数

**UI**：
- 火焰图标 + 连续天数显示
- 火焰大小随天数变化（1天小火 → 7天中火 → 30天大火）

### 4.3 成就徽章系统

**数据库变更**：`user_config` 表新增键
- `earned_badges` - JSON 数组存储获得的徽章

**徽章类型**（8个起步）：
- 🌅 早起学者 - 连续 7 天在 6:00 前开始
- 🔥 一周坚持 - 连续 7 天有学习记录
- ⚡ 闪电突击 - 单日学习超过 8 小时
- 🎯 目标达成 - 完成周学习目标
- 🌙 夜猫子 - 累计深夜（23:00-5:00）学习 10 小时
- 🏆 百日磨剑 - 累计学习 100 小时
- 📚 千锤百炼 - 累计学习 1000 小时
- 🎪 挑战大师 - 完成任意 30 天挑战

**UI**：
- 设置面板徽章墙展示
- 新徽章获得时的弹窗动画

## Phase 5: 功能扩展 - 实用工具（优先级：中）

### 5.1 数据导出功能

**文件**：`main.js` + `preload.js` + `renderer.js`
**功能**：
- CSV 导出：适合 Excel 二次分析
- JSON 导出：适合备份和程序处理
- 导出文件位置通过 Electron `dialog.showSaveDialog()` 选择

### 5.2 学习标签分类系统

**数据库变更**：
```sql
-- 标签表
CREATE TABLE study_tags (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL UNIQUE,
  color TEXT DEFAULT '#007AFF',
  created_at TEXT DEFAULT CURRENT_TIMESTAMP
);

-- Session-标签关联表
CREATE TABLE session_tags (
  session_id INTEGER,
  tag_id INTEGER,
  PRIMARY KEY (session_id, tag_id),
  FOREIGN KEY (session_id) REFERENCES study_sessions(id),
  FOREIGN KEY (tag_id) REFERENCES study_tags(id)
);
```

**功能**：
- 开始学习时可选择/创建标签
- 标签管理（增删改、颜色选择）
- 历史记录按标签筛选
- 按标签统计学习时长

### 5.3 番茄钟集成

**功能**：
- 可切换普通计时 / 番茄钟模式
- 默认配置：25min 工作 + 5min 休息，4 轮后长休息 15min
- 阶段切换时的系统通知和提示音
- 番茄完成数量统计

## 验证方案

每个阶段完成后需进行以下验证：

1. **功能测试**：
   - 开始/结束学习流程正常
   - 历史记录可查看、可删除
   - 新功能操作流程通顺
   - 数据库操作无错误

2. **兼容性测试**：
   - 开发模式 (`npm run start`) 正常
   - 打包后 (`npm run dist`) 路径和数据库正常
   - 旧数据迁移成功无丢失

3. **性能测试**：
   - 窗口启动时间 < 1 秒
   - 学习中 CPU 占用 < 1%
   - 内存占用稳定，无明显泄漏

4. **UX 验证**：
   - 动画流畅无卡顿
   - 界面响应即时（按钮点击反馈 < 100ms）
   - 小窗口模式下所有元素可点击、可阅读
