# 学习时间记录应用 - 用户体验优化实施计划

## Context

本计划旨在对已有完整功能的 Electron 学习时间记录应用进行用户体验优化和设计系统规范化。应用已具备核心功能，但在新用户引导、状态可视化反馈、设计系统一致性等方面存在改进空间。

**目标**：遵循 web-design-engineer 设计原则，通过增量改进提升整体用户体验，保持现有功能完整性与性能。

---

## 实施范围（按优先级）

### 🔴 高优先级（第一阶段）

#### 1. 设计系统变量补充与规范化

**问题**：缺少警告色系变量、间距档位不完整、圆角档位缺失、动画时长未标准化。

**实施方案**：
- 在 `:root` 中补充 `--warning` 色系变量（橙色系）
- 补充 `--sp-0`、`--sp-0_5`、`--sp-7`、`--sp-8` 间距变量
- 补充 `--radius-xs`、`--radius-md` 别名一致性
- 补充动画时长规范：`--duration-xs`/`--sm`/`--md`/`--lg`
- 添加 `--overlay` 遮罩透明度变量
- 添加 `--success-rgb`、`--danger-rgb` 用于 rgba 混合

**修改文件**：`styles.css` (L6-L52)

**复用模式**：直接扩展现有 CSS 变量系统，与主题切换机制完全兼容

---

#### 2. 首次使用引导向导弹窗

**问题**：新用户不知道快捷键存在、时间区间概念、各功能入口位置。

**实施方案**：

**DOM 结构**（添加到 `index.html` 弹窗区域末尾）：
```html
<div id="onboarding-modal" class="modal-root">
  <div class="modal-backdrop"></div>
  <div class="modal-panel modal-center modal-sm onboarding-panel">
    <div class="onboarding-icon">👋</div>
    <div class="onboarding-title">欢迎使用学习时间记录</div>
    <div class="onboarding-content">
      <div class="onboarding-step">
        <span class="step-number">①</span>
        <span>点击「开始记录」或按 <kbd>Ctrl+Alt+S</kbd></span>
      </div>
      <div class="onboarding-step">
        <span class="step-number">②</span>
        <span>需要休息时按 <kbd>Ctrl+Alt+P</kbd> 暂停</span>
      </div>
      <div class="onboarding-step">
        <span class="step-number">③</span>
        <span>完成后按 <kbd>Ctrl+Alt+E</kbd> 结束记录</span>
      </div>
      <div class="onboarding-hint">
        更多功能在右上角设置按钮中探索 🎯
      </div>
    </div>
    <label class="onboarding-checkbox">
      <input type="checkbox" id="onboarding-dont-show" checked>
      <span>下次启动不再显示</span>
    </label>
    <div class="onboarding-footer">
      <button id="onboarding-start" class="primary">开始使用</button>
    </div>
  </div>
</div>
```

**CSS 样式**：
- 玻璃态面板效果，加大圆角
- 步骤列表垂直排列，数字与文字对齐
- `<kbd>` 标签使用 monospace 字体，浅色背景边框
- 复选框使用原生样式 + 自定义标签

**JavaScript 逻辑**（添加到 `renderer.js`）：
- 应用启动时检查 `onboarding_shown` 配置
- 未显示过则自动展示向导弹窗
- 勾选"不再显示"则写入配置持久化
- 点击"开始使用"关闭弹窗，不做其他操作

**修改文件**：
- `index.html` - 添加向导弹窗 DOM
- `styles.css` - 新增向导弹窗样式类
- `renderer.js` - 引导逻辑与配置读写

#### 3. 暂停状态视觉强化

**问题**：暂停状态反馈不够强烈，迷你模式下辨识度低。

**实施方案**：

**正常模式增强**：
- 计时器数字添加柔和的透明度脉冲动画（`opacity` 0.72 → 0.9，1.5s 循环）
- 暂停按钮边框改为警告色（橙色），添加轻微外发光
- "已暂停"文字标签前添加暂停图标前缀

**迷你模式增强**：
- 整个迷你窗口边框添加橙色呼吸光效（box-shadow 动画）
- 计时器前显示小型暂停图标
- 百分比数字降低权重到 0.6 opacity

**系统托盘菜单**：
- 在 `main.js` 托盘菜单中，暂停的会话标注"[暂停中]"状态

**修改文件**：
- `index.html` - 新增暂停图标元素
- `styles.css` - 脉冲动画 keyframes、边框光效样式
- `main.js` - 托盘菜单状态更新逻辑
- `renderer.js` - 暂停状态更新函数扩展

#### 5. 时间区间预设选项

**问题**："05:00-05:00"含义不直观，缺乏快捷预设。

**实施方案**：

**DOM 扩展**（`time-range-modal` body 区域）：
```html
<div class="time-range-presets">
  <label class="preset-option">
    <input type="radio" name="time-preset" value="natural">
    <span class="preset-label">自然日</span>
    <span class="preset-value">00:00 - 24:00</span>
  </label>
  <label class="preset-option">
    <input type="radio" name="time-preset" value="student" checked>
    <span class="preset-label">学习日</span>
    <span class="preset-value">05:00 - 05:00</span>
  </label>
  <label class="preset-option">
    <input type="radio" name="time-preset" value="workday">
    <span class="preset-label">工作日</span>
    <span class="preset-value">08:00 - 22:00</span>
  </label>
  <label class="preset-option">
    <input type="radio" name="time-preset" value="custom">
    <span class="preset-label">自定义</span>
    <span class="preset-value">手动设置</span>
  </label>
</div>
```

**JavaScript 逻辑**：
- 选中预设时自动填充对应的时间输入框
- 手动修改输入框时自动切到"自定义"选项
- 当前配置对应的预设自动高亮

**修改文件**：
- `index.html` - 预设选项 DOM
- `styles.css` - 单选预设卡片样式
- `renderer.js` - 预设选择逻辑绑定

#### 6. 数据可视化悬浮 Tooltip

**问题**：热力图与柱状图无数值提示，难以精确读取。

**实施方案**：

**Tooltip DOM**（复用单例模式，添加到 body 末尾）：
```html
<div id="chart-tooltip" class="chart-tooltip hidden">
  <div class="tooltip-date"></div>
  <div class="tooltip-value"></div>
</div>
```

**CSS 样式**：
- 绝对定位，跟随鼠标位置
- 深色半透明背景，白色文字
- 8px 圆角，内边距 8px 12px
- 0.15s 淡入淡出过渡

**JavaScript 逻辑**：
- 柱状图：`mouseenter` 时显示，`mousemove` 更新位置（偏移鼠标 6px），`mouseleave` 隐藏
- 热力图：同上，每个 cell 绑定事件
- 位置计算确保 tooltip 不超出视口边界
- 内容格式化：日期 + "学习 X 小时 X 分"

**修改文件**：
- `index.html` - tooltip 容器
- `styles.css` - tooltip 样式与动画
- `renderer.js` - 鼠标事件绑定与位置计算

## 实施顺序建议

1. **先做设计系统变量补充** — 为后续所有 UI 改动奠定基础
2. **暂停状态视觉强化** — 改动最小，收益最直接
3. **首次使用引导** — 独立模块，不影响现有功能
4. **设置面板分组** — DOM 结构调整，需谨慎测试
5. **时间区间预设** — 复用现有弹窗逻辑
6. **数据可视化 tooltip** — 纯 UI 增强，独立可测
