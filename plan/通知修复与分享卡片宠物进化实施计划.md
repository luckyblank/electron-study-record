# 通知修复 + 分享卡片 + 宠物进化树 实施计划

> 实施日期：2026-06-27
> 状态：已完成

---

## 一、背景与目标

### 问题 1：2 小时通知不弹

用户反馈学习超过 2 小时后没有收到系统通知。逐条排查发现两处根因：

1. **Windows 通知未注册 AppUserModelId**：`new Notification(...)` 在 Windows 11 下，如果没有事先调用 `app.setAppUserModelId(appId)`，Action Center 会把通知归到 `electron.exe` 名下，开发模式直接静默丢弃，打包后也会受 Toast Push 限制。
2. **`notified_over_2h_state` 用错日期 key**：[renderer.js:371](../renderer.js#L371) 旧版用 `new Date()` 取的自然日 key，但 `calculateTodaySeconds()` 走的是统计区间（默认 `05:00-05:00`）。当用户在凌晨 0:00-5:00 学习时，自然日已切换但统计日还是昨天，导致 `state.date !== todayKey` 误判 → `count` 被重置 → 触发逻辑紊乱。

### 问题 2：连续学习天数必须结束记录才统计

[main.js:617](../main.js#L617) `getEndedSessionRows()` 明确 `WHERE end_time IS NOT NULL`，所有连续天数判定都基于此。后果：今天点了"开始"但还没"结束"，今天**不计入**连续天数，会反向制造焦虑。

### 新功能 1：每日/每周总结分享卡片

把学习数据一键生成可分享的图片，保存到桌面。让用户能把成就发到朋友圈/群聊，转化为正向激励。

### 新功能 2：宠物进化树

当前 3 只宠物只有等级数字变化，没有外观演进。引入"阶段（Stage）"概念，让宠物在等级 10 / 30 / 50 触发可见的外观变化（光环、眼镜、学位帽等），强化"成长"叙事。

---

## 二、设计原则

1. **零新依赖**：所有新功能用现有技术栈（Electron Notification、Canvas、SVG、CSS）实现
2. **不破坏既有数据**：宠物进化只是 UI 层映射 `level → stage`，不需要数据库迁移
3. **遵循现有架构**：所有 OS/DB 交互走 `ipcMain.handle` → `preload.contextBridge` → `window.studyRecord.*`
4. **主题适配**：分享卡片背景/文字色读取当前主题（light / dark / eyecare）

## 三、修复方案

### 3.1 Windows AppUserModelId

在 [main.js](../main.js) `app.whenReady().then(...)` 第一行注册：

```js
if (process.platform === 'win32') {
  app.setAppUserModelId('com.luckyblank.studytimer')  // 必须与 package.json build.appId 一致
}
```

### 3.2 通知失败可观测

[main.js:1432-1452](../main.js#L1432-L1452) `app:notify` 改写：

- 先 `Notification.isSupported()` 防御
- 监听 `notif.on('failed', ...)`
- 失败/异常都通过 `electron-log` 落盘到 `%APPDATA%/study-time-record/logs/main.log`

### 3.3 maybeNotifyHourly 用统计日 key

[renderer.js:367-396](../renderer.js#L367-L396) 改用主进程的 `statsTodayBoundaries()` 返回值的 `start` 作为日期键，与累计时长口径完全一致。

### 3.4 连续天数包含进行中 session

[main.js:646-674](../main.js#L646-L674) 重写 `getStudyStatDays()`：

- 改用全表查询，不再 `WHERE end_time IS NOT NULL`
- 对 `end_time IS NULL` 的行临时构造一个 `effectiveRow`：`end_time` 用"现在"，`paused_duration` 用 `estimatePausedDurationForOngoing(row)` 补齐"暂停起点到现在"的差值
- 仅用于"是否打卡"判定，不写回数据库

这意味着用户今天点了开始，即使没点结束，连续天数也会立即 +1。

## 四、新功能 1：分享卡片

### 数据流

```
┌─────────────┐   ipc: share:get-summary    ┌──────────────┐
│  Renderer   │ ──────────────────────────► │  Main        │
│  (canvas)   │ ◄────────────────────────── │  (聚合数据)  │
└──────┬──────┘    { totalSeconds, tags,    └──────────────┘
       │             dayBuckets, quote… }
       │ canvas.toDataURL('image/png')
       │ ipc: share:save-image (dataUrl, defaultName)
       ▼
┌──────────────────────────────────────────┐
│ Main 写入桌面 → 失败回退 Save As 对话框  │
└──────────────────────────────────────────┘
```

### 4.1 主进程

[main.js](../main.js) 新增两个 IPC handler：

| handler | 入参 | 出参 |
|---|---|---|
| `share:get-summary` | `{ scope: 'day' \| 'week', dayOffset }` | `{ totalSeconds, sessionCount, streak, tags[], dayBuckets[], quote, rangeLabel }` |
| `share:save-image`  | `{ dataUrl, defaultName }` | `{ success, path }` 或 `{ success: false, canceled }` |

聚合逻辑复用现有 `computeRangeBoundsForOffset` / `getSessionSecondsInBounds`，标签时长用"session 内标签数均分"的简化策略（不引入新维度的精确归因）。

### 4.2 渲染端

[renderer.js](../renderer.js) `shareCardManager` 模块：

- **画布尺寸**：720×960（3:4 适合朋友圈竖图）
- **背景**：线性渐变 + 两个径向圆斑装饰，颜色读自当前主题
- **结构（从上至下）**：
  1. 顶部：品牌 + 日期范围
  2. 大标题（带渐变）
  3. 寄语引号
  4. 三宫格核心数据（学习时长 / 专注次数 / 连续天数）
  5. 可变区：
     - **today** → 标签分布条形图（最多 6 个，按时长降序）
     - **week** → 7 天柱状图
  6. 底部品牌slogan

- **保存**：`canvas.toDataURL('image/png')` → 主进程 → 桌面 `study-day-summary_<timestamp>.png`

### 4.3 UI 入口

[index.html](../index.html) 设置面板新增一张卡片「分享卡片」，弹窗 `#share-modal` 包含：

- Day/Week tabs（切换 scope）
- 等宽 canvas 预览
- 「换一张」（重新拉数据，主要为了换寄语）
- 「保存到桌面」

### 4.4 主题适配

`getTheme()` 在 canvas 绘制前读 `document.body.getAttribute('data-theme')`，按三档返回不同的色板，确保黑色主题下文字能看清。

## 五、新功能 2：宠物进化树

### 5.1 阶段定义

| Stage | 等级阈值 | 装饰 |
|---|---|---|
| 0 幼年 | Lv 1-9   | 体型缩小到 0.86 |
| 1 成长 | Lv 10-29 | + 腮红 |
| 2 学者 | Lv 30-49 | + 圆框眼镜 |
| 3 大师 | Lv 50+   | + 学位帽 + 金色光环 + drop-shadow |

阶段映射函数：

```js
function getPetStage(level) {
  if (level >= 50) return 3
  if (level >= 30) return 2
  if (level >= 10) return 1
  return 0
}
```

### 5.2 PET_META 扩展

每只宠物新增 `stageNames: [string, string, string, string]`：

- 学霸猫：初生奶猫 → 机敏花猫 → 博学虎斑 → 玄学神猫
- 努力汪：好奇幼犬 → 勤奋牧犬 → 机警柴犬 → 智勇双全
- 智慧鸮：毛茸幼鸮 → 夜读雏鸮 → 深思学者 → 通晓贤者

### 5.3 SVG 渲染改造

[renderer.js](../renderer.js) `renderPetSvg(petId, stage = 0)` 改造：

- 接受 `stage` 参数
- 通过 SVG 内层 `<g>` 条件拼接：halo / cap / glasses / blush / scale 变换
- 不改变 viewBox，保证现有 CSS 选择器（`.pet-svg`、`.pet-body-group`）继续生效

`PetManager.render()` 在每次重渲染时根据当前等级算 stage 传入，同时把 stage 写到 `dataset.stage` 方便 CSS 选择器（如 `.pet-svg.pet-stage-3 .pet-body-group { filter: drop-shadow(...) }`）

### 5.4 进化触发

[renderer.js](../renderer.js) `PetManager.addExp()` 比较 `prevStage / newStage`：

- 阶段提升时调用 `this.render()` 强制重画 SVG
- 调用 `showEvolutionToast()` 触发 `pet-visual.is-evolving` 炫光动画（CSS keyframes `petEvolveGlow` 2.2s）
- 弹粒子 + Toast `✨ 进化为「XXX」!`

### 5.5 进化树可视化

`renderCollection()` 在每张宠物卡片下方追加 `.pet-evolution-tree`：

- 4 个迷你头像横向排列，箭头 `›` 分隔
- 已达成阶段 → `opacity: 1, filter: grayscale(0)`
- 未达成 → `opacity: .42, filter: grayscale(.8)`
- 当前阶段 → `transform: scale(1.08)` + 边框 + accent 阴影

这样用户在宠物弹窗里就能看清完整成长路径。
